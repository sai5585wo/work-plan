# Python/Pandas 数据报表自动化项目面试讲解稿

## 1. 30 秒介绍

我做了一个跨境电商 ERP 数据报表自动化 Demo。

这个项目用 mock CSV 模拟商品、订单、库存、补货和异常数据，再用 Python/Pandas 自动完成 SKU 检查、订单汇总、补货建议和异常汇总，最后生成运营、采购和客服可以查看的报表。

它的重点不是写复杂系统，而是把原本需要人工筛选、复制、汇总的表格流程，整理成可自动运行的数据报表流程。

## 2. 1 分钟介绍

这个项目模拟的是跨境电商日常运营中的 ERP 数据处理场景。

我先用飞书多维表格理解业务对象，包括商品资料、订单记录、库存、补货采购和异常记录。然后把这些业务数据整理成 mock CSV，再用 Python/Pandas 做自动化处理。

项目里主要完成了几类报表：

- SKU 一致性检查，确认订单、库存、补货和异常表中的 SKU 都能匹配商品资料表。
- 订单汇总，按 SKU 和平台统计销量、销售额和订单数量。
- 补货建议，根据缺口数量筛选需要跟进的 SKU，并关联商品资料表带出供应商和成本。
- 异常汇总，统计异常类型和处理状态，并通过影响订单号关联订单表，生成未关闭异常清单和处理建议。

这个项目体现的是：我能把业务流程拆成数据处理步骤，再用 Python 把重复表格工作自动化。

## 3. 3 分钟介绍

这个项目的背景是跨境电商运营中，经常会有商品、订单、库存、补货和异常数据分散在不同表格里。

如果完全靠人工处理，会有几个问题：

- SKU 是否一致要手动核对。
- 订单销量和销售额要手动汇总。
- 低库存和缺货 SKU 要手动筛选。
- 补货时要在补货表和商品资料表之间来回查供应商和成本。
- 异常订单要人工查影响订单号、平台和处理状态。

所以我用 mock CSV 模拟了 5 张核心业务表：

- 商品资料表
- 订单记录表
- 库存表
- 补货采购表
- 异常记录表

然后用 Python/Pandas 做了几类自动化报表。

第一类是 SKU 检查。商品资料表是 SKU 主表，其他业务表里的 SKU 都要能匹配它，否则后面订单、库存、补货和异常都会对不上。

第二类是订单汇总。我按 SKU 汇总销量和销售额，也按平台汇总订单和销售额，同时筛选待处理和异常订单。这对应运营常见的日报统计。

第三类是补货建议。我从补货采购表中筛选有缺口、系统建议补货、且没有取消的 SKU，再通过 SKU 关联商品资料表，带出供应商和成本，计算预计补货成本，生成采购可以看的补货建议表。

第四类是异常汇总。我从异常记录表中统计异常类型和处理状态，筛选未关闭异常，再通过影响订单号关联订单表，带出平台和订单状态，并根据异常类型生成处理建议。

这个项目最后产出的不是单纯代码，而是一组业务报表。它可以导入飞书多维表格，给运营、采购、客服协作处理。

我认为这个项目体现了三个能力：

- 第一，能理解业务表之间的关系，比如 SKU 连接商品，影响订单号连接异常和订单。
- 第二，能用 Python/Pandas 把业务规则自动化，比如筛选、汇总、关联、计算。
- 第三，能把结果整理成业务人员能看的报表，而不是只停留在代码层面。

## 4. 面试常见问题与回答

### 问题 1：这个项目的数据是真实的吗？

回答：

不是。当前使用的是 mock CSV 数据，目的是保护真实订单和客户隐私，同时练习跨境电商 ERP 的业务流程。

真实工作中，这些数据可以来自 Amazon、Temu、Shopee、TikTok Shop 后台导出，也可以来自 ERP、仓库系统或飞书多维表格导出。

### 问题 2：为什么要做 SKU 检查？

回答：

因为 SKU 是商品维度的连接字段。

订单表、库存表、补货表和异常表中的 SKU，都需要能在商品资料表中找到。如果 SKU 对不上，后面的产品名、供应商、成本、库存和异常都会匹配错误。

所以 SKU 检查是后续所有报表可信的前置条件。

### 问题 3：为什么补货建议表要关联商品资料表？

回答：

补货采购表能告诉我哪个 SKU 缺货、缺多少，但它不一定完整包含供应商和成本。

商品资料表里有供应商和成本。通过 SKU 把两张表关联后，补货建议表就能告诉采购：

```text
这个 SKU 缺多少
找哪个供应商
单件成本是多少
预计补货成本是多少
```

这样补货建议才是可执行的。

### 问题 4：为什么异常表要关联订单表？

回答：

异常记录表告诉我有什么异常，但不一定能完整说明这个异常影响哪张平台订单。

所以我用：

```text
异常记录表.影响订单号 = 订单记录表.订单号
```

把异常和订单关联起来。这样可以看到异常影响的订单、平台、订单状态和负责人，方便后续处理。

### 问题 5：这个项目和飞书有什么关系？

回答：

飞书更适合展示、协作和状态跟进；Python 更适合批量处理数据。

我的理解是：

```text
Python 负责把原始 CSV 自动清洗、汇总、关联、计算
飞书负责承接报表，让团队筛选、查看、分配和更新状态
```

所以它们不是互相替代，而是配合使用。

### 问题 6：这个项目有什么业务价值？

回答：

它减少了人工在 Excel / 飞书里反复筛选、复制、汇总和匹配的工作。

比如：

- 自动检查 SKU 是否一致。
- 自动汇总订单和销售额。
- 自动筛选需要补货的 SKU。
- 自动关联供应商和成本。
- 自动生成未关闭异常清单和处理建议。

这些都是运营、采购、客服日常会反复做的表格工作。

回答：

我会先保证报表逻辑稳定，然后再考虑扩展：
