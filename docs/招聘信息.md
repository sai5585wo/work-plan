## 2. 求职推论与能力边界数据

通过上述数据输入，结合 `我的天赋.md` 和 `个人规划.md`，反推得出当前求职方向的三个关键判断：

1.  **先懂业务，再谈自动化：** 岗位虽然叫 AI 自动化工程师，但真实痛点先落在商品、订单、库存、物流、客服、报表和状态流转上。没有业务流程理解，直接谈 Python、RPA 或模型没有说服力。

## 3. 新策略修正：从 Listing 小工具转向 AI 自动化工程师作品集

当前判断：

1.  Listing 半自动生成工具只能证明基础学习能力，不足以形成求职或副业竞争力。
2.  更值得深挖的是跨境企业反复出现的 ERP、订单、库存、广告、报表、状态流转和数据清洗问题。
3.  当前主线不是副业服务，而是 AI 自动化工程师求职作品集。表格整理服务只能作为辅助验证，不作为当前核心目标。

### 记录模板

```text
公司名称：
所在区域：
岗位名称：
薪资范围：
平台/业务：
提到的工具：
涉及的数据对象：

暴露的痛点：
可转化作品：
可转化服务：
需要补齐的能力：
优先级判断：
```

字段说明：

- `提到的工具`：例如 ERP、店小秘、赛盒、Excel、飞书、影刀等。
- `涉及的数据对象`：例如商品、SKU、订单、库存、广告、售后、物流等。
- `核心工作流程`：例如上架、订单处理、库存更新、补货、报表统计。
- `暴露的痛点`：例如重复录入、数据分散、报表人工整理、库存不准。
- `可转化作品`：例如 ERP 流程模拟表、库存预警看板、订单处理状态表、订单/库存报表自动化脚本。
- `可转化服务`：例如商品/订单/库存表格整理、基础飞书工作台搭建。该项是辅助判断，不是当前主线。
- `需要补齐的能力`：例如 Python、Pandas、RPA、SQL、Coze、Agent、LLM API 等。
