# 3-Day Learning Plan: Feishu Base Workflow Practice

> 中文说明：这份计划用于用 3 天系统练习飞书多维表格能力。  
> 每天投入：约 4 小时。  
> 练习场景：继续使用当前的跨境商品 Listing 半自动生成表，不额外发散到 Python、API 或复杂自动化。  
> 学习方式：英文术语 + 中文解释，方便你同时熟悉工具词汇。

## Goal（目标）

Use the current cross-border Listing MVP to practice Feishu Base workflow skills.

中文解释：用当前跨境 Listing 表格作为练习项目，系统练习飞书多维表格里的字段、视图、筛选、分组、表单、基础自动化和展示表达。

3 天后你应该完成：

- 一张字段结构清楚、命名稳定、状态明确的飞书多维表格。
- 一个自己操作用的 `Workflow View`。
- 一个适合展示的 `Demo View`。
- 至少 2 个辅助视图：按状态查看、待处理查看。
- 一个商品资料提交表单。
- 一个简单飞书自动化规则。
- 1 条完整测试记录，从表单提交到 AI 生成再回填。
- 一段能用于面试或自我介绍的作品说明。

- [x] Day 2: Views, Filters, and Display Logic（视图、筛选与展示逻辑）

## Day 1: Base Structure and Data Quality（基础表结构与数据质量）

Focus（重点）：让表格不只是“能用”，而是字段清楚、状态清楚、样本数据能支撑后续练习。

Time Plan（4 小时安排）：

- 0.5h：检查字段结构，分清输入字段、输出字段和流程字段。
- 1h：统一字段命名和字段类型，确认单选、公式、文本字段没有混乱。
- 1h：检查 6 条商品样本是否都有完整输入和输出。
- 1h：检查 `生成提示词` 公式是否对新商品仍然可用。
- 0.5h：记录字段规则，避免后续随意新增字段。

Tasks（任务）：

- 输入字段包括：中文品名、产品类目、材质/规格、适用人群、使用场景、核心卖点、目标平台、目标市场。
- 输出字段包括：英文标题、五点描述、搜索关键词、风险提示、人工修改意见、生成时间。
- 流程字段包括：生成状态、生成提示词。
- `目标平台`、`目标市场`、`生成状态` 使用单选字段。
- `生成状态` 至少包含：待生成、已生成、需人工修改、已完成。
- 随机新增 1 条测试商品，只填输入字段，确认 `生成提示词` 能自动生成。

Output（当天产出）：

一张字段结构稳定、样本数据完整、公式可复用的飞书多维表格。

## Day 2: Views, Filters, and Display Logic（视图、筛选与展示逻辑）

Focus（重点）：学会用视图组织工作，而不是把所有信息都堆在一张表里。

Time Plan（4 小时安排）：

- 0.5h：整理当前 `Workflow View`，确认它保留全部字段，适合自己操作。
- 1h：完善 `Demo View`，只展示适合给别人看的字段。
- 1h：创建按状态管理的视图，例如 `To Generate View` 或 `Status View`。
- 1h：练习筛选、分组、字段隐藏、字段排序。
- 0.5h：从“面试官/老板视角”检查展示效果，删除干扰信息。

Tasks（任务）：

- `Workflow View`：保留全部字段，用于你自己操作和检查流程。
- `Demo View`：只保留中文品名、产品类目、目标平台、目标市场、英文标题、五点描述、搜索关键词、风险提示、生成状态。
- `Status View`：按 `生成状态` 分组，练习查看待生成、需人工修改、已完成的记录。
- `To Generate View`：筛选出 `生成状态 = 待生成` 的记录，用来模拟真实待办队列。
- 调整字段顺序，让别人从左到右能看懂：商品 -> 平台/市场 -> 生成结果 -> 风险 -> 状态。
- 检查长文本字段是否影响阅读，必要时在展示视图里隐藏。

Output（当天产出）：

至少 4 个视图：`Workflow View`、`Demo View`、`Status View`、`To Generate View`。你能说清楚每个视图给谁用、解决什么问题。

## Day 3: Form, Feishu Automation, and Portfolio Review（表单、飞书自动化与作品复盘）

Focus（重点）：把表格从“自己手动维护”升级成一个可以演示的小型工作流。

Time Plan（4 小时安排）：

- 1h：创建 Product Submission Form（商品资料提交表单），只收集商品基础信息。
- 1h：设置简单 Feishu Automation（飞书自动化规则），新增记录后自动设置状态。
- 0.75h：用 1 条测试商品跑完整流程：表单提交 -> 生成提示词 -> AI 生成 -> 回填结果 -> 更新状态。
- 0.75h：写作品说明，准备面试或展示话术。
- 0.5h：复盘问题，记录下一步要优化的地方。

Tasks（任务）：

- 表单只收集输入字段，不让提交者填写输出字段和公式字段。
- 自动化规则：新增记录后，把 `生成状态` 设置为 `待生成`。
- 测试商品必须从表单进入，而不是直接在表格里手动新增。
- 测试时确认 `生成提示词` 仍然自动生成。
- AI 输出后，回填英文标题、五点描述、搜索关键词、风险提示。
- 最后把测试记录状态改成 `已完成` 或 `需人工修改`。
- 写 3-5 句话说明这个飞书工作流解决什么问题、怎么运行、下一步能自动化什么。

Output（当天产出）：

一个能演示的飞书小工作流：表单提交商品资料，表格自动生成提示词，AI 生成 Listing，结果回填并用状态管理。

## Key Vocabulary（重点词汇）

- Base（多维表格）：承载业务数据和流程的表格系统。
- Field（字段）：表格中的一列。
- Field type（字段类型）：文本、单选、公式、日期等字段形式。
- View（视图）：同一张表的不同展示方式。
- Filter（筛选）：只显示符合条件的记录。
- Group（分组）：按某个字段把记录分成不同组。
- Form（表单）：给别人提交信息的入口。
- Automation（自动化）：满足条件后自动执行动作。
- Trigger（触发条件）：什么时候启动自动化。
- Action（执行动作）：自动化具体做什么。
- Workflow（工作流）：从输入到输出的一套流程。
- Status（状态）：记录当前进度，例如待生成、已完成。

## Do Not Do Yet（暂时不要做）

这 3 天先不要做这些：

- 不学 Python。
- 不学 API。
- 不做 n8n / Make。
- 不追求全自动。
- 不新增太多商品样本。
- 不把这个 MVP 做成 SaaS。

原因：

> 当前目标是把飞书多维表格的核心能力练熟。先把一个小流程做顺、做清楚，再考虑更复杂的技术。
